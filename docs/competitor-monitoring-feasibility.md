# GEO 项目一页执行版（简化）

## 目标
做一个能让用户持续付费的 GEO 工具：  
`发现问题 -> 自动修复 -> 验证结果 -> 周报提醒`

## 现在只做 4 件事
1. 扫描：robots/sitemap/llms/meta/schema/noindex。
2. 自动修复（安全项）：只改可确定项，支持回滚。
3. 定时任务：每周自动扫描，异常邮件/Webhook。
4. 同行监控（MVP）：关键词下你站 vs 竞品的可见性变化。

## 先不做
- 小时级实时监控
- 一期多语言全覆盖
- 复杂企业权限系统
- 全渠道无边界抓取

## 两周执行计划

### 第 1 周
- 完成扫描结果统一模型（pass/warn/fail + fixable）。
- 完成自动修复队列（dry run / apply / rollback）。
- 完成每周扫描与通知模板。

### 第 2 周
- 上线同行监控 MVP（10 竞品、100 关键词、周更新）。
- 输出 3 个固定结果：
  - 你站与竞品可见性对比
  - 新增/丢失引用来源
  - 本周可执行修复清单

## 最大暗坑（只看这 3 个）
1. 数据来源不稳定或不合规  
对策：优先官方 API/合规供应商，保留原始快照。

2. 自动修复误伤站点  
对策：只做安全项，默认 dry run，必须可回滚。

3. 成本失控  
对策：按套餐限制关键词数、竞品数、扫描频率。

## 验收标准（够用就行）
- 扫描成功率 >= 95%
- 自动修复可回滚率 = 100%
- 每次扫描至少产出 1 条可执行动作
- 同行监控周报可稳定输出

## 商业化（最小可卖）
1. Free：手动扫描 + 基础报告  
2. Pro：定时扫描 + 自动安全修复 + 通知  
3. Agency：多站点 + 导出 + Webhook/API

## 结论
别做大而全。  
先把“自动修复闭环 + 同行监控周报”做稳，这就能卖。
